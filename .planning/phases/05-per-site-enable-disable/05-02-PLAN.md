<objective>
Update all four content scripts to use per-platform enable/disable checks and add re-enable paths so toggling works without page reload.

Purpose: D-04 requires toggling to take effect immediately via `EXTENSION_STATE_CHANGED` message with no tab reload. Currently all four content scripts only handle the disable path (teardown) and rely on page reload for re-enable. This plan adds the re-enable path and switches from `extensionEnabled` to `platformEnabled[platform]`.

Output: All four content scripts check per-platform state and handle both disable and re-enable via messaging.
</objective>
