<task type="auto">
  <name>Task 1: Install ws dependency and create mock WebSocket server</name>
  <files>tests/fixtures/mock-ws-server.ts</files>
  <action>
Run: `npm install --save-dev ws @types/ws`

Create `tests/fixtures/mock-ws-server.ts` with:
- Import `WebSocketServer, WebSocket` from `ws`
- Module-level `let wss: WebSocketServer | null = null`
- `export async function startMockWsServer(port = 8080): Promise<WebSocketServer>` — creates `new WebSocketServer({ port })`, registers a `connection` handler that:
  - On `message`: parses JSON, if `msg.type === 'ping'` sends back `JSON.stringify({ type: 'pong' })`
  - On `connection`: sends `JSON.stringify({ type: 'connected', data: { overlay_id: 'test-overlay' }, timestamp: new Date().toISOString() })` immediately
- `export async function stopMockWsServer(): Promise<void>` — wraps `wss.close()` in a Promise, resolves on close callback, resolves immediately if wss is null
- Export `export function getMockWsServer(): WebSocketServer | null` for test access

The mock server listens on port 8080 because `src/config.ts` defaults `API_BASE_URL` to `http://localhost:8080`. The extension's service worker opens WebSocket connections to `ws://localhost:8080/ws/chat/{streamer}` in development/test builds.

Do NOT use `page.routeWebSocket()` — it only intercepts page-context WebSockets, not service worker WebSockets.
