## Spacing Scale

Declared values from existing `@theme` block in `src/ui/styles.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--spacing-1`) | Icon gaps, inline padding, button icon margins |
| sm | 8px (`--spacing-2`) | Compact element spacing, header gap between right-side buttons |
| md | 16px (`--spacing-4`) | Default element spacing |
| lg | 24px (`--spacing-6`) | Section padding |
| xl | 32px (`--spacing-8`) | Layout gaps |
| 2xl | 48px (`--spacing-12`) | Major section breaks |
| 3xl | 64px (`--spacing-16`) | Page-level spacing |

Exceptions:
- Header buttons: 6px (`py-1.5`) vertical padding matching existing collapse button — preserves existing header height
- Pop-out and "Switch" buttons in header: minimum 28px touch target height (existing header height constraint)
- "Chat popped out" indicator banner: 8px (`--spacing-2`) vertical padding, 12px (`--spacing-3`) horizontal padding — matches existing connection-failed banner pattern
- "Switch to AllChat" injected button in native pop-out: 8px vertical, 12px horizontal — must not expand native chat chrome

Source: existing header layout `px-2 py-1.5` from `ChatContainer.tsx`.

## Typography

All roles use Inter (variable font stack). DM Mono reserved for monospace/code content only. No new type sizes are introduced — only the existing 4 sizes from the token set are used.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (messages, banner text) | 14px (`--text-base: 0.875rem`) | 400 | 1.5 |
| Label (buttons, badges, small status text) | 13px (`--text-sm: 0.8125rem`) | 400 | 1.2 |
| Emphasis (usernames, action labels) | 13px (`--text-sm: 0.8125rem`) | 600 | 1.2 |
| Caption (xs status, empty state sub-text) | 11px (`--text-xs: 0.6875rem`) | 400 | 1.4 |

## Color

All values are OkLCh semantic tokens from `src/ui/styles.css`. No new colors are introduced for this phase.

| Role | Token | Computed value | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `--color-bg` → `--color-neutral-950` | `oklch(0.09 0.007 270)` | Full iframe background, pop-out window background |
| Secondary (30%) | `--color-surface` → `--color-neutral-900` | `oklch(0.11 0.009 270)` | Header bar, "Chat popped out" banner, "Switch to AllChat" button bg |
| Elevated surface | `--color-surface-2` → `--color-neutral-800` | `oklch(0.14 0.008 270)` | Hover state for injected "Switch to AllChat" button |
| Accent (10%) | Platform color var e.g. `--color-twitch` | `#A37BFF` / `#FF4444` / `#53FC18` | Platform badge dot in header only |
| Destructive | `bg-red-900/50`, `border-red-700` | existing | Connection-failed banner only (existing pattern, unchanged) |

Accent reserved for:
- The 8px platform-colored dot in the chat header (existing)
- The left border on "Switch to AllChat" injected button (2px platform color, matching `.platform-*` class pattern)

Do NOT apply accent to:
- Pop-out button icon
- "Chat popped out" banner
- "Switch to native" button
- Any hover or focus states (use `--color-text` / `--color-text-dim` transition)

Button default state: `text-[var(--color-text-dim)] hover:text-text transition-colors` — identical to existing collapse button and consistent with all header icon buttons.

"Chat popped out" banner background: `bg-[var(--color-surface)] border-b border-[var(--color-border)]` with `text-[var(--color-text-sub)]` — surface-level, not a warning/error color.

"Switch to AllChat" injected button in native chat: background `var(--color-surface)`, border `1px solid var(--color-border-md)`, text `var(--color-text)`. No platform accent on the button itself — only InfinityLogo SVG carries the brand.

Source: pre-populated from `ChatContainer.tsx` existing patterns and `src/ui/styles.css` tokens.

## Interaction States

### Pop-out button (header)

| State | Visual |
|-------|--------|
| Default | `color: var(--color-text-dim)` |
| Hover | `color: var(--color-text)`, `transition-colors 150ms` |
| Active (pop-out window open) | Same as default — no persistent active indicator |
| Disabled (platform disabled via per-site toggle) | Does not render — content script never injects AllChat when platform is disabled (D-17) |

### "Switch to native" button (header)

| State | Visual |
|-------|--------|
| Default | `color: var(--color-text-dim)` |
| Hover | `color: var(--color-text)`, `transition-colors 150ms` |
| In-page mode: click | Hides AllChat iframe, restores native chat. AllChat button appears in native chat. |
| Pop-out mode: click | Navigates pop-out window to platform native chat URL |

### "Chat popped out" indicator (in-page)

| State | Visual |
|-------|--------|
| Visible | Fills full iframe area. Surface background. Centered content. |
| "Bring back" button | `bg-[var(--color-surface-2)] hover:bg-[var(--color-neutral-700)] text-text text-sm px-3 py-1.5 rounded transition-colors` |

### "Switch to AllChat" injected button

| State | Visual |
|-------|--------|
| Default | Surface background, border, InfinityLogo + label |
| Hover | `background: var(--color-surface-2)` |
| Click | Replaces native chat content with `chat-container.html` in same window |

## Icon Contract

All icons are inline SVG at 16×16px (`w-4 h-4`). No external icon library.

| Icon | Element | SVG shape |
|------|---------|-----------|
| Pop-out (external link arrow ⇗) | Pop-out button | `<path d="M10 6L18 6M18 6L18 14M18 6L8 16M4 4h6v2H6v10h10v-4h2v6H4V4z" .../>` at 24×24 viewBox, scaled to `w-4 h-4`. Use a standard external-link path: square with arrow pointing out of top-right corner. |
| Left chevron or X | "Switch to native" button | Use minimal text label instead of icon if icon would add visual noise: `< Native` in 11px (`text-xs`) caps. Or use a left-arrow SVG (`←`) at `w-3 h-3`. |

Decision for "Switch to native": use text label `Native` with a leading `←` SVG (12×12px, `w-3 h-3`) placed left of text. Label is 11px (`text-xs`) weight 400. This matches the compact header space and is self-explanatory without hover tooltip dependency.

## Layout Constraints

### Header layout (left → right)

```
[←collapse] [InfinityLogo centered] [connection-dot] [platform-badge] [←native] [⇗pop-out]
```

- Total right-side cluster: `flex items-center gap-1.5` — existing gap, do not change
- "Switch to native" and pop-out button are added to the right cluster, maintaining `gap-1.5`
- All right-side elements remain in a single `flex items-center gap-1.5` wrapper

### "Chat popped out" indicator layout

```
[centered vertically in iframe]
  InfinityLogo (32px)
  "Chat is open in a separate window"  (14px, --color-text-sub)
  "Your chat is running in the pop-out window."  (13px, --color-text-dim)
  [Bring back] button
```

- Container: `flex flex-col items-center justify-center h-full gap-3`
- Same structure as existing empty state in messages container

### Pop-out window default dimensions

Before any persisted size exists: `width=420, height=700` — matches the typical chat panel width on streaming platforms.

Persisted keys in `chrome.storage.local`: `popout_window_width`, `popout_window_height`, `popout_window_x`, `popout_window_y` (all integers, pixels).
