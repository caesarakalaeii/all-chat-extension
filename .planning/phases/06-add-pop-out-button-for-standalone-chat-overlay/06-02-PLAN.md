**3. Chrome runtime Port for pop-out mode:**

Replace the existing `window.addEventListener('message', handleMessage)` effect with a dual-mode approach. In the main message listening effect:

```typescript
useEffect(() => {
  if (isPopOut) {
    // Pop-out mode: use chrome.runtime.Port for direct SW communication
    const port = chrome.runtime.connect({ name: POPOUT_PORT_NAME });
    
    port.onMessage.addListener((message: Record<string, unknown>) => {
      // Handle CONNECTION_STATE and WS_MESSAGE same as postMessage handler
      handleIncomingMessage(message);
    });
    
    // Handle port disconnect (SW restart) — reconnect with backoff
    port.onDisconnect.addListener(() => {
      console.warn('[AllChat UI PopOut] Port disconnected, reconnecting...');
      // Retry connection after 1s
      setTimeout(() => {
        const newPort = chrome.runtime.connect({ name: POPOUT_PORT_NAME });
        newPort.onMessage.addListener((message: Record<string, unknown>) => {
          handleIncomingMessage(message);
        });
        // Re-request connection state
        chrome.runtime.sendMessage({ type: 'GET_CONNECTION_STATE' }).then((response) => {
          if (response?.success) {
            handleIncomingMessage({ type: 'CONNECTION_STATE', data: response.data });
          }
        });
      }, 1000);
    });
    
    // Connect websocket via direct sendMessage (not postMessage)
    chrome.runtime.sendMessage({ type: 'CONNECT_WEBSOCKET', streamerUsername: streamer });
    chrome.runtime.sendMessage({ type: 'GET_CONNECTION_STATE' }).then((response) => {
      if (response?.success) {
        handleIncomingMessage({ type: 'CONNECTION_STATE', data: response.data });
      }
    });
    
    return () => port.disconnect();
  } else {
    // In-page iframe mode: use existing window.parent.postMessage relay
    window.parent.postMessage({ type: 'GET_CONNECTION_STATE' }, '*');
    
    const messageHandler = (event: MessageEvent) => {
      handleIncomingMessage(event.data);
    };
    window.addEventListener('message', messageHandler);
    
    return () => window.removeEventListener('message', messageHandler);
  }
}, [isPopOut, streamer]);
```

Extract the existing message handling logic into a `handleIncomingMessage(data: Record<string, unknown>)` function. This function receives the message data (not the event) and handles CONNECTION_STATE, WS_MESSAGE, LOGIN_SUCCESS, LOGIN_ERROR, POPOUT_OPENED, POPOUT_CLOSED.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iframe ↔ content script | postMessage with `*` origin (existing pattern; extensionOrigin enforced on content script side) |
| pop-out ↔ service worker | chrome.runtime.Port (extension-only, no cross-origin risk) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02 | Spoofing | POPOUT_REQUEST message from iframe | accept | Existing extensionOrigin check on content script message listener already validates origin; POPOUT_REQUEST follows same pattern |
| T-06-03 | Tampering | messages array in POPOUT_REQUEST | mitigate | Content script writes to chrome.storage.local (sandboxed); pop-out reads from same sandboxed storage. Cap at POPOUT_MAX_MESSAGES (50) to prevent storage abuse |
| T-06-04 | Denial of Service | Rapid pop-out button clicks | mitigate | Content script only opens one pop-out window; subsequent POPOUT_REQUEST ignored if pop-out already open |
</threat_model>
