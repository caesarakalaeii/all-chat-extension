## Critical Gaps

### Gap 1: YouTube WebSocket Relay Broken (Production Defect)

**Affects:** YouTube E2E flow
**Severity:** Critical — silent functional defect in production

`src/content-scripts/youtube.ts` line 227:
```ts
const iframes = document.querySelectorAll('#allchat-iframe');
```

`PlatformDetector.injectAllChatUI()` never sets `id="allchat-iframe"` on the iframe — it sets `data-platform` and `data-streamer` attributes. Twitch and Kick correctly use `iframe[data-platform="twitch"]` and `iframe[data-platform="kick"]`. YouTube's relay is stale from a prior iteration.

**Effect:** On YouTube, `CONNECTION_STATE` and `WS_MESSAGE` messages from the service worker find 0 iframes and are silently dropped. Chat iframe renders but never receives live chat or connection status.

**Fix:** Change `youtube.ts` line 227: `'#allchat-iframe'` → `'iframe[data-platform="youtube"]'`

---

### Gap 2: KICK-05d Test Fails on Every CI Run

**Affects:** CI green status
**Severity:** Critical — blocks CI

`tests/test-postmessage-origin.spec.ts` KICK-05d (lines 46–59) asserts that `src/ui/index.tsx` contains a `window.addEventListener('message')` with `event.origin !== extensionOrigin` guard. The current `index.tsx` (32 lines) uses URL-param initialization — no message listener exists. This is the correct and more secure architecture (URL params vs postMessage for init), but the test was not updated when the design shifted.

**Fix:** Update KICK-05d to assert the URL-param pattern (`?platform=...&streamer=...`) instead of the removed message listener, or remove the test if it's fully superseded.
