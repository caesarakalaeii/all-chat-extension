<task type="auto">
  <name>Task 1: Install / upgrade packages and delete tailwind.config.js</name>
  <files>package.json</files>
  <action>
Run the following commands in sequence from the repo root:

```bash
# Install Tailwind 4 stack + fontsource + MiniCssExtractPlugin
npm install tailwindcss@^4.2.1 @tailwindcss/postcss mini-css-extract-plugin @fontsource/inter @fontsource/dm-mono

# Upgrade tailwind-merge to v3 (required for Tailwind 4 class name semantics)
npm install tailwind-merge@^3.0.0

# Remove autoprefixer (Tailwind 4 handles vendor prefixes natively)
npm uninstall autoprefixer
```

Then delete `tailwind.config.js`:
```bash
rm tailwind.config.js
```

Verify the deletions and installs:
```bash
ls tailwind.config.js 2>/dev/null && echo "ERROR: still exists" || echo "OK: deleted"
node -e "console.log(require('./node_modules/tailwindcss/package.json').version)"
node -e "console.log(require('./node_modules/tailwind-merge/package.json').version)"
```

Do NOT run npx @tailwindcss/upgrade codemod at this stage — that modifies
styles.css and component files which are handled in plans 02-03 and 02-04.
The codemod would conflict with the targeted rewrites in those plans.


**webpack.config.js** — make three targeted changes:

1. Add require at top of file (after existing requires):
```javascript
const MiniCssExtractPlugin = require('mini-css-extract-plugin');
```

2. Replace the CSS rule (the `test: /\.css$/` rule):
```javascript
{
  test: /\.css$/,
  use: [MiniCssExtractPlugin.loader, 'css-loader', 'postcss-loader']
},
{
  test: /\.(woff|woff2|eot|ttf|otf)$/i,
  type: 'asset/resource',
  generator: {
    filename: 'ui/fonts/[name][ext]'
  }
}
```

3. Add MiniCssExtractPlugin to the plugins array (before the existing CopyPlugin line):
```javascript
new MiniCssExtractPlugin({
  filename: 'ui/chat-styles.css'
}),
```

this point. The build will FAIL until Plan 02-03 updates styles.css to use
`@import "tailwindcss"`. That is expected. Run the build here only to confirm
the webpack plugin configuration is syntactically correct (not to verify the
full CSS pipeline — that comes in plan 02-03).
