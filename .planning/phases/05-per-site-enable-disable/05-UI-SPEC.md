## Interaction Contract

### Toggle behavior
- **On toggle**: immediate optimistic UI update (checked state changes before async write completes)
- **On error**: revert toggle state, show no special UI (silent revert matches existing `handleToggle` pattern)
- **Disabled state**: `disabled={isLoading}` on all three checkboxes during initial load (matches existing pattern)
- **Transition**: CSS `transition: 0.3s` on toggle slider (reuse existing)

### Icon state (extension toolbar)
- **Platform enabled**: color icon (`assets/icon-{size}.png`) set per-tab via `chrome.action.setIcon({ tabId, path })`
- **Platform disabled**: grayscale icon (`assets/icon-{size}-gray.png`) set per-tab
- **Non-platform tab**: color icon (default — no override needed)
- **Update triggers**: (1) popup toggle fires → update icon for all tabs of that platform; (2) `SET_CURRENT_PLATFORM` message in service worker → set icon for sender tab based on platform enabled state

### Active row highlight
- Read `current_platform` from `chrome.storage.session` on popup mount
- Apply `.platform-row--active` to the matching row
- If `current_platform` is `null` or undefined, no row is highlighted

## Icon Design Contract

### Grayscale icon assets (new files)
- `assets/icon-16-gray.png` — 16×16 desaturated version of `icon-16.png`
- `assets/icon-32-gray.png` — 32×32 desaturated version of `icon-32.png`
- `assets/icon-48-gray.png` — 48×48 desaturated version of `icon-48.png`
- `assets/icon-128-gray.png` — 128×128 desaturated version of `icon-128.png`

Generation: Apply CSS filter `grayscale(100%) brightness(0.7)` equivalent in an image editor, or use ImageMagick: `convert icon-{n}.png -colorspace Gray icon-{n}-gray.png`

Rationale: Pre-generated PNGs chosen over `OffscreenCanvas` at runtime — service workers have constrained DOM environment and pre-generated assets are simpler and robust. (CONTEXT.md Claude's Discretion — RESEARCH.md Pattern 5 recommendation)
