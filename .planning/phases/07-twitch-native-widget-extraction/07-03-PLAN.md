**1. Add tabBarMode state:**

Add a new state variable:
```typescript
const [tabBarMode, setTabBarMode] = useState(false);
```

**2. Listen for TAB_BAR_MODE message:**

In the existing `useEffect` that sets up the message handler (the `handleIncomingMessage` function), add handling for `TAB_BAR_MODE`:

```typescript
if (data.type === 'TAB_BAR_MODE') {
  setTabBarMode(data.enabled as boolean);
  return;
}
```

This must be added inside the `handleIncomingMessage` function, before the existing `CONNECTION_STATE` check.

**3. Modify header rendering:**

Replace the current header JSX block. When `tabBarMode` is `true`:
- Do NOT render the header div (`<div className="px-2 py-1.5 bg-surface border-b...">`)
- Do NOT render: collapse button, InfinityLogo, connection dot, platform badge, switch-to-native button
- DO render the pop-out button, but position it absolutely in the top-right corner of the iframe:

```tsx
{tabBarMode && !isPoppedOut && !isPopOut && (
  <div style={{ position: 'absolute', top: 4, right: 4, zIndex: 10 }}>
    <button
      onClick={handlePopOut}
      className="text-[var(--color-text-dim)] hover:text-text transition-colors bg-surface/80 rounded p-1"
      title="Open in new window"
      aria-label="Open chat in new window"
    >
      <svg className="w-4 h-4" fill="none" stroke="currentColor" viewBox="0 0 24 24">
        <path strokeLinecap="round" strokeLinejoin="round" strokeWidth={2} d="M18 13v6a2 2 0 01-2 2H5a2 2 0 01-2-2V8a2 2 0 012-2h6M15 3h6v6M10 14L21 3" />
      </svg>
    </button>
  </div>
)}
```

When `tabBarMode` is `false` (YouTube, Kick, pop-out, or no tab bar):
- Render the full existing header exactly as-is (no changes)

The conditional structure should be:

```tsx
{/* Header — hidden when tab bar controls view (Twitch) */}
{!tabBarMode && (
  <div className="px-2 py-1.5 bg-surface border-b border-border flex items-center">
    {/* ... existing full header content unchanged ... */}
  </div>
)}

{/* Pop-out button (floating) when in tabBarMode — still accessible from iframe */}
{tabBarMode && !isPoppedOut && !isPopOut && (
  <div style={{ position: 'absolute', top: 4, right: 4, zIndex: 10 }}>
    {/* pop-out button SVG */}
  </div>
)}
```

**4. Remove handleSwitchToNative calls from iframe UI in tabBarMode:**

When `tabBarMode` is true, the "Switch to native" button is already removed (it's part of the header). The `handleSwitchToNative` function can remain — it's still used in pop-out mode (`isPopOut` navigates to native pop-out URL).

**5. The wrapping `<div className="h-full flex flex-col bg-bg">` must gain `position: relative`** to allow the absolute pop-out button positioning:

```tsx
<div className="h-full flex flex-col bg-bg" style={{ position: 'relative' }}>
```

Or use the Tailwind class `relative`:
```tsx
<div className="h-full flex flex-col bg-bg relative">
```

**6. Do NOT change the isCollapsed logic.** When `tabBarMode` is true, the collapse button is gone, so `isCollapsed` will always be `false` (its initial value). The collapsed state only matters for non-tabBarMode contexts.
