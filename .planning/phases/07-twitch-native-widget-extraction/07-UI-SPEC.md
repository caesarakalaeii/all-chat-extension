# Phase 7 — UI Design Contract: Twitch Native Widget Extraction

> Visual and interaction contract for the tab bar, widget zones, and extracted Twitch widgets.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind 4 + OkLCh tokens) |
| Preset | not applicable |
| Component library | none — hand-rolled components, no Radix/Base-UI |
| Icon library | inline SVG (no external icon library) |
| Font (iframe) | Inter 400/600 + DM Mono 400 via @fontsource |
| Font (tab bar) | Inter 400/600 via inline style referencing page stack fallback — see Tab Bar note below |

**Tab Bar note:** The tab bar is injected by the content script into the platform page DOM (outside the iframe). It cannot use the iframe's `chat-styles.css` bundle. Styling is applied via inline `style` attributes referencing hardcoded OkLCh/hex values that match the design tokens in `src/ui/styles.css`. No external font load is triggered — Inter is not loaded on Twitch pages, so the tab bar font stack falls back to system-ui with Inter as the preferred value: `font-family: Inter, system-ui, sans-serif`.

Source: CONTEXT.md D-05, RESEARCH.md Standard Stack.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tab bar internal padding, widget zone min-padding |
| sm | 8px | Tab button horizontal padding, widget zone internal padding |
| md | 16px | Default element spacing, tab bar total height (36px = sm*2 + icon + text) |
| lg | 24px | Section gaps inside widget zones |
| xl | 32px | Layout gaps between major zones |
| 2xl | 48px | Reserved — not used in this phase |
| 3xl | 64px | Reserved — not used in this phase |

**Tab bar height:** 36px total (8px top + 8px bottom padding + ~20px content). This is the `padding-top` offset applied to `#allchat-container` so the iframe does not underlap the tab bar.

**Widget zone sizing:** `flex: 0 0 auto` — height determined by widget content. Empty zones collapse to `height: 0; overflow: hidden`. A present widget expands the zone to its natural height. No min-height imposed.

Exceptions: Tab bar touch target height is 36px, which satisfies WCAG 2.5.5 minimum 24px (AAA 44px is a best-effort recommendation; 36px is acceptable for desktop-only extension context).

Source: RESEARCH.md Pattern 1, Pattern 2.

## Color

All values reference the established OkLCh token system from `src/ui/styles.css`. Tab bar uses the equivalent hardcoded hex/oklch values since it cannot import the CSS bundle.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--color-bg` = `oklch(0.09 0.007 270)` ≈ `#07070a` | Iframe background, widget zone backgrounds |
| Secondary (30%) | `--color-surface` = `oklch(0.11 0.009 270)` ≈ `#0d0d12` | Tab bar background, active tab indicator background |
| Accent (10%) | `--color-twitch` = `#A37BFF` | Active tab bottom border (AllChat tab only when active), InfinityLogo stroke |
| Destructive | not applicable | No destructive actions in this phase |

**Tab bar color breakdown:**

| State | Element | Value |
|-------|---------|-------|
| Tab bar background | Container | `oklch(0.11 0.009 270)` (#0d0d12) — matches `--color-surface` |
| Tab bar bottom border | 1px separator | `oklch(from #fff l c h / 0.06)` — matches `--color-border` |
| Active tab | Bottom border indicator | 2px solid `#A37BFF` (`--color-twitch`) |
| Active tab text | Label | `oklch(0.91 0.003 270)` (#e8e8ec) — matches `--color-neutral-100` / `--color-text` |
| Inactive tab text | Label | `oklch(0.58 0.007 270)` (#8c8c96) — matches `--color-neutral-400` / `--color-text-sub` |
| Tab hover background | Tab button | `oklch(0.14 0.008 270)` — matches `--color-surface-2` |

**Widget zones:**

| Zone | Background | Border |
|------|------------|--------|
| Top zone (`#allchat-widget-zone-top`) | transparent (Twitch native widget renders its own bg) | none — widget provides its own chrome |
| Bottom zone (`#allchat-widget-zone-bottom`) | transparent | 1px top border `--color-border` only when a channel points widget is present |

**Accent reserved for:**
- Active tab bottom border indicator (2px, AllChat tab only)
- InfinityLogo animated SVG stroke in the AllChat tab label

The accent color (`#A37BFF`) is NOT applied to: general text, buttons, hover states, widget zone borders, or inactive tab elements.

Source: CONTEXT.md D-05, `src/ui/styles.css` token definitions.

## Component Inventory

New components introduced in this phase:

### 1. Tab Bar (`#allchat-tab-bar`) — DOM element, not React

**Location:** Injected directly into `.chat-shell` by `twitch.ts`. Lives outside the iframe.

**Structure:**
```
#allchat-tab-bar                    (position: absolute; top: 0; left: 0; right: 0; z-index: 2; height: 36px)
├── #allchat-tab-allchat            (flex button — left half of bar)
│   ├── [InfinityLogo SVG 16px]     (inline SVG, stroke: #A37BFF)
│   └── [Text: "AllChat"]           (13px, weight 600, Inter)
└── #allchat-tab-twitch             (flex button — right half of bar)
    └── [Text: "Twitch Chat"]       (13px, weight 600, Inter)
```

**Layout:** `display: flex; width: 100%`. Each tab button is `flex: 1`. Separator between tabs: `1px solid oklch(from #fff l c h / 0.06)` right border on the AllChat tab.

**Active state:** Bottom border `2px solid #A37BFF` on the active tab button.

**Inactive state:** No bottom border. Text color uses `--color-text-sub` value.

**Transition:** `color 0.15s ease` on tab text. No transform/scale animations — WCAG respects `prefers-reduced-motion`, so no decorative motion.

### 2. Top Widget Zone (`#allchat-widget-zone-top`) — DOM element

**Location:** First child of `#allchat-container` (below tab bar, above iframe wrapper).

**Behavior:** `flex: 0 0 auto; overflow: hidden; max-height: 0` by default. When a transient widget (prediction, poll, hype train, raid) is cloned into it, `max-height` is removed and `overflow` is set to `visible`.

**Contents:** Raw Twitch widget DOM clones. The zone provides no visual chrome — Twitch widgets render their own backgrounds.

### 3. Bottom Widget Zone (`#allchat-widget-zone-bottom`) — DOM element

**Location:** Last child of `#allchat-container` (below iframe wrapper).

**Behavior:** `flex: 0 0 auto; overflow: hidden`. Contains the channel points widget clone.

**When populated:** 1px top border `oklch(from #fff l c h / 0.06)` is applied to visually separate the widget from the iframe.

### 4. Iframe Wrapper (`#allchat-iframe-wrapper`) — DOM element

**Location:** Middle child of `#allchat-container`, between the two widget zones.

**Behavior:** `flex: 1 1 0; min-height: 0`. Contains the existing `#allchat-iframe` element. This wrapper takes all remaining vertical space after widget zones consume their natural height.

## Layout Contract

```
.chat-shell  (position: relative — already set by Phase 1 injection)
├── #allchat-tab-bar                (position: absolute; top: 0; left: 0; right: 0; z-index: 2; height: 36px)
└── #allchat-container              (position: absolute; inset: 0; padding-top: 36px; z-index: 1; display: flex; flex-direction: column)
    ├── #allchat-widget-zone-top    (flex: 0 0 auto; overflow: hidden; max-height: 0 by default)
    ├── #allchat-iframe-wrapper     (flex: 1 1 0; min-height: 0)
    │   └── #allchat-iframe         (width: 100%; height: 100%; border: none)
    └── #allchat-widget-zone-bottom (flex: 0 0 auto; overflow: hidden)
```

**When "Twitch Chat" tab is active:**
- `#allchat-container` → `display: none`
- Native chat CSS hide style (`#allchat-hide-native-style`) is removed from DOM
- Tab bar remains at `z-index: 2` with tab bar background covering top 36px

**When "AllChat" tab is active (default):**
- `#allchat-container` → visible (flex column)
- Native chat CSS hide style is present
- Widget zones populated from cloned native elements

Source: RESEARCH.md Pattern 2, CONTEXT.md D-03, D-06, D-07.

## Interaction States

### Tab Switching

| Trigger | Action | Duration |
|---------|--------|---------|
| Click "AllChat" tab | Show `#allchat-container`, re-inject native chat hide style, update active tab indicator | Instant (no animation — DOM show/hide) |
| Click "Twitch Chat" tab | `#allchat-container` → `display: none`, remove native chat hide style, update active tab indicator | Instant |

No slide/fade transitions on the view swap — instant toggle, matching Twitch's own tab behavior (native feel per CONTEXT.md phase goal).

### Widget Clone States

| Widget Type | Appearance trigger | Removal trigger | Visual feedback |
|------------|-------------------|-----------------|-----------------|
| Channel points (persistent) | Init (cloned once) | Teardown | None — always present for logged-in users |
| Prediction / Poll / Hype Train | MutationObserver detects element in native DOM | MutationObserver detects removal | None — clone appears/disappears with original |
| Raid banner | MutationObserver detects element | MutationObserver detects removal | None |

Widget clones do not animate in/out. They appear when the original appears in the native DOM. The native Twitch widget itself may have its own entrance animation — that animation is part of the clone and will play normally.

### Tab Bar Focus / Keyboard

| Interaction | Behavior |
|-------------|---------|
| Tab key | Standard browser focus order through tab buttons |
| Enter / Space | Activates focused tab (same as click) |
| Focus ring | `outline: 2px solid #A37BFF; outline-offset: -2px` on `:focus-visible` |

## Copywriting Contract

| Element | Copy |
|---------|------|
| AllChat tab label | "∞ AllChat" (∞ rendered as InfinityLogo SVG 16px inline, followed by " AllChat" text) |
| Twitch Chat tab label | "Twitch Chat" |
| AllChat tab aria-label | "AllChat tab — cross-platform chat view" |
| Twitch Chat tab aria-label | "Twitch Chat tab — native Twitch chat view" |
| Tab bar container aria-label | "Chat view switcher" |
| Widget zone top aria-label | "Twitch interactive widgets — predictions, polls, hype trains" |
| Widget zone bottom aria-label | "Twitch channel points" |
| Widget clone aria-hidden | true — clones are presentational; interactivity is delegated to originals |
| Empty top zone state | No visible copy — zone height is 0, invisible |
| Empty bottom zone state | No visible copy — zone collapses when no channel points widget present (logged-out viewers) |
| Error: clone sync failure | No user-facing copy — failure is silent (clone may be stale until next MutationObserver tick) |

**No primary CTA button exists in this phase.** The interaction model is tab switching + cloned widget interaction. No form submissions, no confirm dialogs.

**No destructive actions.** Tab switching is reversible and non-destructive. Widget clones are read-through to native Twitch elements which have their own destructive flows (e.g., redeeming channel points) — those flows are owned by Twitch's native UI and are not modified.

Source: CONTEXT.md D-04, D-13; auto-mode defaults for accessibility copy.

## ChatContainer Header Changes

Phase 7 removes the AllChat iframe header and the "Switch to native" button from `ChatContainer.tsx`.

| Element removed | Replacement |
|----------------|-------------|
| `<div className="px-2 py-1.5 bg-surface border-b...">` (header div) | Tab bar in content script DOM |
| InfinityLogo in header center | InfinityLogo inline SVG in AllChat tab label |
| "Switch to native" button | Twitch Chat tab in tab bar |
| Collapse button | Removed — not needed when tab bar controls view switching |
| Connection dot | Moved to inside the AllChat tab label (right of "AllChat" text, 6px dot) |
| Platform badge | Removed — tab bar context makes platform obvious |
| Pop-out button | Remains in-iframe — position shifts to top-right corner with no surrounding header padding |

The ChatContainer receives a `tabBarMode: boolean` prop (sent via `postMessage` from content script on init) that removes header rendering when `true`. When `tabBarMode` is false (other platforms, pop-out mode), the existing header renders as-is — no regression on YouTube/Kick/pop-out.

Source: CONTEXT.md D-04, D-08, code_context section.

## Accessibility Notes

- Tab bar buttons use `role="tab"` and `aria-selected="true/false"` per ARIA tab pattern.
- Tab bar container uses `role="tablist"` with `aria-label="Chat view switcher"`.
- Widget zones use `role="region"` with descriptive `aria-label` attributes.
- Widget clones are `aria-hidden="true"` — screen reader users interact with the originals in the hidden native chat (which remains in DOM at `visibility: hidden`, not `display: none`).
- All interactive tab bar elements meet 24px minimum touch target height (36px actual).
- Color contrast: inactive tab text `oklch(0.58...)` on `oklch(0.11...)` background = ~4.6:1 (WCAG AA pass). Active tab text `oklch(0.91...)` on `oklch(0.11...)` = ~8.2:1 (WCAG AAA pass).
